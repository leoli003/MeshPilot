## 功能说明

一步生成实施计划，跳过常规计划模式的确认步骤。生成计划后直接输出到对话中，供用户审查和修改。

## 使用方式

```
/mesh-quick-plan <任务描述>
```

或直接描述要规划的内容：
```
帮我规划一个用户认证系统
```

## 工作流程

1. **分析任务** - 理解用户想要实现的内容
2. **选择合适的代理** - 根据任务性质选择:
   - `ecc:planner` - 复杂功能、架构决策、重构
   - `Plan` - 一般规划任务
   - `ecc:architect` - 系统设计、扩展性、技术决策
   - 默认使用 `ecc:planner`
3. **生成计划** - 使用选定的代理创建完整计划
4. **输出供审查** - 将计划呈现给用户确认/修改

## 代理选择逻辑

| 任务类型 | 代理 |
|---------|------|
| 功能实现、重构 | ecc:planner |
| 系统架构、扩展性 | ecc:architect |
| 一般规划、简单任务 | Plan |

## 输出格式

```markdown
# 计划: [任务名称]

## 概述
[将要实现的内容简述]

## 步骤
1. [步骤 1]
   - 详情: ...
   - 文件: ...
2. [步骤 2]
   ...

## 依赖
- [需要的依赖项]

## 风险
- [需要注意的潜在问题]

## 预估工作量
- [时间/复杂度估算]
```

## 计划生成后

计划输出后:
1. 提示用户: "请确认这个计划，或告诉我需要修改的地方"
2. 等待用户回复
3. 如需修改，相应更新计划
4. 确认后，用户可以开始实施

## 示例

用户: `/mesh-quick-plan 添加用户登录功能`

输出:
```markdown
# 计划: 用户登录功能

## 概述
实现基于 JWT 的用户认证系统，包含登录、登出、token 刷新功能。

## 步骤
1. 创建认证模块结构
   - 文件: src/auth/, src/auth/types.ts
2. 实现 JWT token 管理
   - 文件: src/auth/jwt.ts
3. 创建登录 API 端点
   - 文件: src/api/auth.ts
4. 添加前端登录表单
   - 文件: src/components/LoginForm.tsx
5. 集成认证中间件
   - 文件: src/middleware/auth.ts

## 依赖
- jsonwebtoken 包
- bcrypt 用于密码哈希

## 风险
- Token 存储安全性
- 密码传输加密

## 预估工作量
- 中等: 2-3 天
```

请确认这个计划，或告诉我需要修改的地方。

## 注意事项

- 此技能专为速度设计 - 无中间确认
- 计划是起点 - 用户随时可以修改
- 复杂任务时，选定的代理可能进行额外探索
- 保持计划可执行且具体
